Question 2: HTML Elements
1. Five Elements I Found Most Challenging to Implement
While developing the VYLINE Airport Pickups & Shuttle Services website, several HTML elements were challenging because I am still building my experience .

a) <nav> – Creating a navigation menu that correctly linked to different sections such as Services, Booking, and Contact was difficult at first. I had to learn how anchor links work.
b) <section> – I initially struggled to decide how to divide the webpage into meaningful sections like airport transfers, shuttle services, and company information.
<form> – Designing the booking form was challenging because I needed to understand input fields, labels, and form structure.


c) <img> – Images of vehicles and airport pickups did not display correctly  due to incorrect file paths.


d) <footer> – I was unsure what information belonged in the footer, but later included contact details, operating hours, and company information.
These challenges helped me better understand webpage structure and functionality.


2. Use of Semantic Elements to Structure Content
I used semantic HTML elements to organize the VYLINE website professionally:
<header> contained the company name, logo, and slogan.
<nav> provided links to important sections like Home, Services, Booking, and Contact.
<section> separated major content areas such as airport pickups, shuttle services, pricing, and booking information.
<article> was used to describe individual services offered by VYLINE.
<footer> displayed contact information, social media links, and copyright details.
Using semantic elements made the website easier to read, maintain, and understand.



3. Most Useful Element for Organizing Layout
The <div> element was the most useful for organizing my layout. It allowed me to group related content such as service cards, booking details, and contact information. 



Question 3: HTML Attributes
1. Three Essential Attributes
The three attributes that were essential for making the  website functional were:
href – Used in navigation links so users could move between sections or access booking information.
src – Used to display images of airport shuttles, vehicles, and service illustrations.
alt – Provided alternative descriptions for images, improving accessibility and ensuring users understood content if images failed to load.



2. Use of Class and ID Attributes
I used the class attribute to group similar elements, such as service cards for airport pickups, shuttle transfers, and private transport options. This helped maintain consistent organization.
The id attribute was used for unique sections like the booking form and contact section so navigation links could scroll directly to those areas.


3. Attribute that Improved User Experience the Most
The alt attribute improved user experience the most because it ensured accessibility. If images of VYLINE vehicles or services failed to load, users could still understand the content. It also supports screen readers, making the website more inclusive.
